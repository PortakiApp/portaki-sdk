# Schéma SQL — module train (référence)

Les tables **`t_e_module_train_*`** sont créées par Flyway dans **`portaki-api`** (`V41__module_train_cache.sql`, etc.). Ce dossier documente la **convention de nommage** alignée sur le reste du produit :

- **`t_e_`** — tables d’entité
- **`t_j_`** — tables de jointure

Les fichiers `.sql` ici peuvent servir de **copie de référence** ; l’historique Flyway effectif reste sous `portaki-api/.../db/migration/`.
